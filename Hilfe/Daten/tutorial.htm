Schnelleinstieg


Um die giDX² Bibliothek verwenden zu können sollten Sie Grundkenntnisse in C/C++ haben und sich mit
dem Visual C++ Compiler von Microsoft auskennen. Das Einarbeiten in die Funktionsweise von giDX² ist
verhältnismäßig einfach zu erlernen. Ich habe versucht die Verwendung von Funktionen so einfach wie
möglich zu gestallten. Ohne viel Vorarbeit sollte es möglich sein DirectX zu initialisieren oder ein
Bild zu laden. Die giDX² Bibliothek kümmert sich auch um die Speicherverwaltung für Bitmaps und
Sounddateien. Dieser Schnelleinstieg soll dem interessierten Anwender eine Möglichkeit geben die
wichtigsten giDX²-Funktionen kennenzulernen. An einem kleinen Programm werde ich Ihnen zeigen wie
man ein Bitmap ladet, es mit den Pfeiltasten bewegt auf Kollision prüft und eine Sounddatei abspielt.

Initialisierung


Den größten Teil der Initialisierung übernimmt die giDX² Engine. Sie müssen nur eingeben welche Auflösung
sie haben wollen und ob die Anwendung im Fenster- oder Vollbildmodus laufen soll. Es ist auch möglich eine
laufende Anwendung zwischen Vollbild und in einem Fenster darzustellen.


#include<gidx.h>

int main()
{
// Initialisiert DirectX7
//
// GXFULLSCREEN Vollbild
// GXDEBUG Fenster
if(graphic(800,600,32,GXDEBUG)!=0)
return(GXRETURN);


while(1)
{
// Aktualisiere Tastaturabfrage
statkey();
// Wurde ESC gedrückt?
if(keyhit(DIK_ESCAPE))return (GXRETURN);
}
return(GXRETURN);// Wichtig! Beim verlassen, GXRETURN zurückgeben
}


Bitmap laden und zeichnen


giDX² kann Bilder im bmp-Format laden. Die Bildinformationen werden in der Struktur IMG gespeichert.
Der Anwender muss sich um die Struktur IMG nicht kümmern. Er übergibt der Funktion einen Zeiger auf diese
Struktur und giDX² füllt diese mit den nötigen Informationen. Wurde das Bitmap erfolgreich geladen kann
mit einem einzigen Funktionsaufruf das Bitmap in den Bildschirmspeicher geschrieben werden.


#include<gidx.h>

int main()
{
    // Speicher für die Bitmaps
    LPIMAGE bild1;
    LPIMAGE bild2;

    // Initialisiert DirectX7
    //
    // GXFULLSCREEN        Vollbild
    // GXDEBUG                     Fenster
    if(graphic(800,600,32,GXDEBUG)!=0)
           return(GXRETURN);

    // Bitmaps in laden
    loadimage("bild1.bmp",&bild1,0);
    loadimage("bild2.bmp",&bild2,0);

    while(1)
    {
       // Bilder in den Bildschirmspeicher kopieren
       drawimage(384,284,bild1,GXCLIPPING);
       drawimage(384,50,bild2,GXCLIPPING);

       // Aktualisiere Tastaturabfrage
       statkey();
       // Wurde ESC gedrückt?
       if(keyhit(DIK_ESCAPE))return (GXRETURN);

       // Bildschirmspeicher auf den Monitor
       // GXFLIP_WAIT wartet bis das Bild volständig  aufgebaut ist
       flip(GXFLIP_WAIT);
    }

return(GXRETURN);// Wichtig! Beim verlassen, GXRETURN zurückgeben
}


Bitmap über die Tastatur bewegen


Auch die Tastaturabfrage beschränkt sich auf das Nötigste. Eine Initialisierung ist hier nicht nötig.
Mit wenigen Zeilen Code erfolgt die Abfrage der Tastatur. Wir erweitern das Beispiel und bewegen das
in den Speicher geladene Bild mit den Cursortasten auf dem Monitor.


#include<gidx.h>

int main()
{
    // Speicher für die Bitmaps
    LPIMAGE bild1;
    LPIMAGE bild2;

    int x=384;
    int y=50;

    // Initialisiert DirectX7
    //
    // GXFULLSCREEN        Vollbild
    // GXDEBUG                     Fenster
    if(graphic(800,600,32,GXDEBUG)!=0)
           return(GXRETURN);

    // Bitmaps in laden
    loadimage("bild1.bmp",&bild1,0);
    loadimage("bild2.bmp",&bild2,0);

    // Farbe definieren, später verwendet cls() diese Farbe um den Monitor zu löschen
    color(0,0,0);

    while(1)
    {
       // Aktualisiere Tastaturabfrage
       statkey();

       // Tastenabfragen
       if(keydown(DIK_UP))
               y=y-1;
       if(keydown(DIK_DOWN))
               y=y+1;
       if(keydown(DIK_RIGHT))
               x=x+1;
       if(keydown(DIK_LEFT))
                       x=x-1;

       // keydown prüft ob die Taste gedrückt ist
       // keyhit prüft ob die Taste einmal gedrückt wurde
       if(keyhit(DIK_ESCAPE))return (GXRETURN);

       // Bildschirm löschen mit schwarz
       cls();

       // Bilder in den Bildschirmspeicher kopieren
       drawimage(384,284,bild1,GXCLIPPING);
       drawimage(x,  y,  bild2,GXCLIPPING);

       // Bildschirmspeicher auf den Monitor
       // GXFLIP_WAIT wartet bis das Bild volständig  aufgebaut ist
       flip(GXFLIP_WAIT);
    }

return(GXRETURN);// Wichtig! Beim verlassen, GXRETURN zurückgeben
}


Kollisionsabfrage


Die Idee einer einfach zu verwendenden Game Engine kommt auch bei der Kollisionsabfrage zum tragen.
Ein einfacher Funktionsaufruf prüft ob sich zwei Bitmaps überschneiden.


Soundausgabe


Wie bei der Tastaturabfrage ist auch bei der Soundausgabe eine Initialisierung nicht nötig. Die Funktionen
sind ahlich wie beim laden von Bitmaps. Auch hier werden die Informationen in einer Struktur gespeichert.
Und mit einem Funktionsaufruf abgespielt.


#include<gidx.h>

int main()
{
     // Speicher für die Bitmaps
    LPIMAGE bild1;
    LPIMAGE bild2;

    // Speicher für Sound
    LPSOUND snd;

    int x=384;
    int y=50;

    // Initialisiert DirectX7
    //
    // GXFULLSCREEN        Vollbild
    // GXDEBUG                     Fenster
    if(graphic(800,600,32,GXDEBUG)!=0)
          return(GXRETURN);

    // Bitmaps in laden
    loadimage("bild1.bmp",&bild1,0);
    loadimage("bild2.bmp",&bild2,0);

    // Sound laden
    loadsound("error.wav",&snd);

    // Farbe definieren, später verwendet cls() diese Farbe um den Monitor zu löschen
    color(0,0,0);

    while(1)
    {
               // Aktualisiere Tastaturabfrage
       statkey();

       // Tastenabfragen
       if(keydown(DIK_UP))
        y=y-1;
       if(keydown(DIK_DOWN))
          y=y+1;
       if(keydown(DIK_RIGHT))
          x=x+1;
       if(keydown(DIK_LEFT))
          x=x-1;
       if(collision(x,y,bild2,1,384,284,bild1,1))
          play(snd,0,0,0);

       // keydown prüft ob die Taste gedrückt ist
       // keyhit prüft ob die Taste einmal gedrückt wurde
       if(keyhit(DIK_ESCAPE))return (GXRETURN);

       // Bildschirm löschen mit schwarz
       cls();

       // Bilder in den Bildschirmspeicher kopieren
       drawimage(384,284,bild1,GXCLIPPING);
       drawimage(x,  y,  bild2,GXCLIPPING);

       // Bildschirmspeicher auf den Monitor
       // GXFLIP_WAIT wartet bis das Bild volständig  aufgebaut ist
       flip(GXFLIP_WAIT);
    }

return(GXRETURN);// Wichtig! Beim verlassen, GXRETURN zurückgeben
}
