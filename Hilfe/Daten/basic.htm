Basics

Technik

Die giDX² Engine basiert auf dem DirectX7 SDK von Microsoft. Es kapselt Windows und DirectX Funktionen
um die Verwendung zu vereinfachen.

Die Engine besteht aus drei Teilen.


  • giDXWin.lib

  • giDXSDK

  • giDX.h


Den Kern der Engine bilden das giDXSDK. Diese arbeiten direkt mit DirectX zusammen. Die Anwendung
des Entwicklers kommuniziert über giDX.h mit dieser Schnittstelle . Um Flexibel zu bleiben hat der Entwickler
die Möglichkeit den Quellcode des giDXSDK zu verändern oder zu erweitern.


Die Grafikausgabe

giDX² arbeitet mit Double Buffering. Das bedeutet das alle Zeichenoperationen in einem nicht sichbaren Buffer
ausgeführt werden. Das fertige Bild wird mit der Funktion flip( ) in den Frontbuffer kopiert und somit auf dem Monitor
sichtbar. Dadurch wird das Flimmern verhindert, das beim zeichen direkt in den Frontbuffer entstehen würde.

Das Herz der Grafikausgabe ist die Struktur IMG. In dieser werden die Bildinformationen gespeichert.
Die Speicherverwaltung für die Bildinformationen übernimmt giDX². Die Struktur muss nicht von Ihnen befüllt werden.
Das übernimmt giDX² für Sie.

Mit der Funktion loadimage(...) laden Sie ein Bitmap in den Speicher.

int loadimage(const char* file_name, LPLPIMAGE lplpimg, int operate)




Und mit der drawimage(...) Funktion zeichen Sie das Bitmap in den Bildschirmspeicher


int drawimage(int x, int y, LPIMAGE img, int flag)




Die API Spezifikationen

giDX² Anwendungen müssen mit int main( ) beginnen.

#include <windows.h>
#include <gidx.h>

int main()
{
if(graphic(800,600,32,GXDEBUG)!=0)
return(GXRETURN);

return(GXRETURN);
}



Compilereinstellungen für Visual C++ 5.x/6.x


Damit der Visual C++ Compiler die giDX Bibliothek verwenden kann, muß der Ort in dem die giDX² Bibliothek
gespeichert wurde dem Compiler bekannt gemacht werden.
Gehen Sie auf Extras->Optionen und wählen den Reiter Verzeichnisse. Tragen Sie bei den Include-Dateien den
Ort für den Ordner ...gidx/include ein und bei den Bibliothekdateien den Ort für den Ordner ...gidx/lib.
Das DirectX SDK ab Version 7 muss installiert und auch für das DX-SDK die Compilereinstellungen vorgenommen sein.
Obwohl giDX Programme mit int main( ) beginnen ist auch dies eine WIN32 Anwendung.
Um eine neue Anwendung zu beginnen gehen Sie auf Datei->Neu. Wählen Sie Win32-Anwendung. Bestätigen Sie mit OK.
Fügen Sie Ihrem Projekt das giDX² SDK hinzu. In dem Sie bei Projek->Dem Projekt hinzufügen->Datei alle cpp-Dateien im Ordner giDXSDK in Ihre Projekt einfügen. Jetzt können Sie ihren Code eingeben.


Das erste Programm

Erstellen Sie eine neue WIN32 Anwendung in VisualC++. Um die giDX² Funktionen benutzen zu können, muss
die Datei gidx.h in das Programm eingebunden werden.


#include <gidx>


Danach stehen uns die giDX Funktionen zur Verfügung.  giDX² Anwendungen beginnen mit int main(). Dies ist
der Einspringpunkt für die Engine.


#include <windows.h>
#include <gidx.h>

int main()
{
if(graphic(800,600,32,GXDEBUG)!=0)
return(GXRETURN);

return(GXRETURN);
}


Dieses kurze Programm initialisiert DirectDraw mit einer Auflösung von 640x480 im Vollbildmodus.
DirectInput und DirectSound werden von giDX selbständig initialisiert.

Damit die Anwendung nicht gleich beendet wird, bauen wir noch eine Tastaturabfrage ein. Das Programm
soll erst beendet werden wenn ESC gedrückt wird.


#include <windows.h>
#include <gidx.h>

int main()
{
if(graphic(800,600,32,GXDEBUG)!=0)
return(GXRETURN);

while(1)
{
statkey( );
if(keyhit(DIK_ESC))return (GXRETURN);
}
return(GXRETURN);

}


Das Programm wird so lange in der Schleife laufen und den Tastaturbuffer mit statkey( ) aktualisieren und
mit keyhit( ) abfragen bis ESC gedrückt wurde.
